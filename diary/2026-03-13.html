🔍 核心问题：跨 Channel 记忆不一致

发现的大问题：同一个 agent 在不同 channel 记忆不一致

现象：
  • 主 session 记住"超哥爱喝美式咖啡"
  • 飞书 session 查不到这个记忆
  • 反之亦然
根因分析：
  • 不同 channel 使用不同的 userId
  • Mem0 按 userId 隔离记忆
  • OpenClaw 主 session userId = "liuchao"
  • 飞书 session 使用不同的 userId
结论：对 OpenClaw 的记忆系统还需继续深入研究，寻找跨 channel 记忆同步方案
问题未解决 跨 Channel userId 隔离

🤖 AI 失忆问题暴露

有趣的现象：今天更新学习日记时，OpenClaw 忘记了昨天配置好的 GitHub 同步流程，需要提醒"回忆昨天"才会。

问题分析：
  • 每次会话重启后，AI 需要重新读取文件才能知道工作流程
  • Mem0 记忆昨天刚配置，内容还不完整
  • 真正的持久化在文件系统，但 AI 不会自动读取所有项目目录
记忆分层架构：
  • Mem0 长期记忆 - 跨会话保留，但还在完善中
  • 文件系统 - 真正的持久化，需要显式读取
  • 会话内存 - 当前对话上下文，重启即失
改进方案：会话启动时除了读 SOUL.md / USER.md / MEMORY.md，还应该自动检查重要的项目目录（如 redcreen.github.io/）
