2. **采购单查询**
   - 张时成采购单：206324（包含NH500-1/2/3/4/5/6等SKU）
   - 支持按7天区间分段查询（API限制）

3. **预入库单创建**
   - 必须基于已有采购单创建（默认规则）
   - 自动生成详细备注格式：
     ```
     供应商，日期，发货地发收货地，共计X包（总数个）
     商品名SKU（数量个）
     ```

**关键规则确立**

| 规则 | 说明 |
|------|------|
| **供应商确认** | 不确定时列出选项，等待确认 |
| **SKU确认** | 不认识或不确定时必须询问 |
| **时间确认** | 预计到货时间必须询问，不自作主张 |
| **二次确认** | 所有写入操作执行前必须二次确认 |
| **例外情况** | 只有明确写"不要确认直接入库"时才跳过 |

**信息解析模式**
```
[供应商] [日期]，[发货地]发[收货地] 共计[X]包（[总数]个）
[商品名][SKU]（[数量]个）
...
```

## 💡 今日心得

1. **规则比代码更重要** - 先确立清晰的业务流程规则，再写代码
2. **确认机制保平安** - 所有写入操作加二次确认，避免误操作
3. **信息解析要严谨** - 不确定就询问，不猜测不假设
4. **备注格式要规范** - 统一的备注格式方便后续查阅
