# 🧠 三德子的记忆系统 v1.0

> 记忆不是奢侈品，是必需品。每一次对话都应该被记录，每一个重要信息都应该被保留。

---

## 核心原则

### 1. 自动记录，绝不遗漏
- 每次对话结束后，自动写入 `memory/YYYY-MM-DD.md`
- 不依赖用户提醒，不依赖"记得保存"
- 如果文件不存在，自动创建

### 2. 分层记忆架构
```
短期记忆（会话内）→ 中期记忆（daily-logs）→ 长期记忆（MEMORY.md）
     ↓                        ↓                      ↓
  当前上下文              当天完整记录            精华萃取
```

### 3. 写入时机
| 场景 | 写入目标 | 触发条件 |
|------|----------|----------|
| 每次回复后 | `memory/YYYY-MM-DD.md` | 自动追加 |
| 重要决策/结论 | `MEMORY.md` | 识别到关键信息 |
| 用户说"记住" | `MEMORY.md` | 显式指令 |
| 任务完成 | `MEMORY.md` + 当天日志 | 里程碑事件 |

## 🔥 重要：智能入库流程（2026-03-15 更新）

**超哥明确要求：在做预约入库时，不清楚SKU的情况下，需要查一下，然后确认！**

### 标准入库流程（批量确认）

```
收到入库信息 
    ↓
1. 解析供应商名称 → 查询供应商候选列表
2. 解析所有商品名称 → 批量查询SKU候选列表
3. 一次性展示所有信息（供应商 + 所有商品 + 仓库）
4. 用户一次性确认（输入格式: 供应商序号,商品1-SKU序号,商品2-SKU序号,...）
5. 创建入库单
```

**确认示例：**
```

============================================================

🏢 【供应商】请选择:
  1. 王小小               | ID: 33533246      | 编码: - 👈 推荐
  2. 王小二               | ID: 13762249      | 编码: -

📦 【商品明细】请为每个商品选择SKU:

  商品1: 黄色回头马 (数量: 100个)
    1. NH500-3         | 黄色回头马               | - 👈 推荐
    2. NH500-3Y        | 黄色回头马-大            | -

  商品2: 红色回头马 (数量: 100个)
    1. NH500-1         | 红色回头马               | - 👈 推荐

============================================================
✏️  请输入确认信息（格式: 供应商序号,商品1-SKU序号,商品2-SKU序号,...）
   示例: 1,1,1 表示：选供应商1，商品1选SKU1，商品2选SKU1
```

### 已实现功能

**1. 商品模糊查询**
- 脚本: `scripts/jushuitan_sku_query.py`
- 用法: `py scripts/jushuitan_sku_query.py "马"`
- 功能: 根据商品名称模糊查询SKU列表

**2. 智能入库脚本（核心）**
- 脚本: `scripts/smart_appointment_in.py`
- 功能: 
  - 自动解析供应商名称 → 查询并确认
  - 自动解析商品描述 → 查询SKU并确认
  - 展示完整信息（供应商、产品、仓库）
  - 用户最终确认后创建入库单

### 关键原则（永久记住）

- ❌ **绝不擅自判断供应商ID**
- ❌ **绝不擅自判断SKU编码**
- ❌ **绝不跳过确认环节（包括自动模式）**
- ✅ **必须先查询，再展示选项，再确认**
- ✅ **能一起确认的就一起确认，减少交互次数**
- ✅ **展示完整的候选列表，让用户一次性选择**
- ✅ **写入性操作必须等用户明确说"确认"后才执行**

**🔴 红线条款：所有写入性操作（创建入库单、创建供应商等）必须等用户明确确认后才执行。不得使用自动模式跳过确认。违反此条=自作主张。**

### 使用示例

```python
# 在程序中调用
from scripts.smart_appointment_in import SmartAppointmentIn

smart = SmartAppointmentIn()
result = smart.create_appointment(
    supplier_name="王小小",           # 会查询并确认
    ship_date="3月15号",
    ship_from="江西",
    ship_to="义乌刘总小马",
    packages=2,
    items_desc=[                     # 每个商品都会查询并确认
        "黄色回头马（100个）",
        "红色回头马（100个）"
    ]
)
```
