⚡ OpenClaw 任务执行机制

深入理解任务并发、资源冲突与解决方案

核心问题

Q:

多个 Agent 的任务在同一个电脑上执行，不会冲突么？比如打开网页订票查询，是排队做，还是可以同时做？

A:

多个 Agent 的任务在同一个电脑上执行，默认是并发的，但可能产生冲突。

任务执行架构图

具体情况分析

1. 浏览器自动化（Playwright/Selenium）

场景 结果 解决方案
两个 Agent 同时操作同一个浏览器实例 ❌ 冲突、报错、行为异常 使用独立浏览器实例
两个 Agent 操作独立的浏览器实例（不同端口/用户数据目录） ✅ 可以并发 配置独立 userDataDir
两个 Agent 同时点击屏幕同一位置 ❌ 冲突 串行执行或虚拟桌面
独立浏览器配置示例 
# Agent A
browser = p.chromium.launch(
    headless=True,
    args=['--user-data-dir=/tmp/agent-a']
)

# Agent B
browser = p.chromium.launch(
    headless=True,
    args=['--user-data-dir=/tmp/agent-b']
)

2. 文件系统操作

场景 结果 说明
Agent A 写 /tmp/data.txt，Agent B 读 ✅ 正常 文件系统支持并发读写
两个 Agent 同时写同一个文件 ⚠️ 数据损坏/覆盖 需要文件锁机制
各自操作自己的工作区文件 ✅ 完全隔离 推荐方式

3. 系统命令（exec）

场景 结果 说明
两个 Agent 同时运行 ls ✅ 正常 只读命令无冲突
两个 Agent 同时安装软件 ⚠️ 可能冲突 包管理器锁会排队
两个 Agent 同时操作同一设备 ❌ 冲突 如同时控制打印机

调度机制图解

默认行为：并发执行

时间轴 →
Agent A:
[==========任务1==========]
Agent B:
[==========任务2==========]
Agent C:
[==========任务3==========]
↑ 三个任务同时执行（并发）

可能的问题

时间轴 →
Agent A:
[点击"搜索"按钮══════►]
Agent B:
[点击"下单"按钮══════►]
⚠️ 同时控制鼠标！
结果：点错位置，任务失败

解决方案对比

方案 1：任务队列（串行化）

任务队列
[任务A] → [任务B] → [任务C]
一个一个执行，避免冲突

方案 2：资源隔离

浏览器 每个 Agent 独立浏览器实例
文件 各自工作区，不共享
屏幕 虚拟桌面 / 容器 / 远程机器
网络端口 各自分配不同端口

方案 3：沙盒/容器（推荐）

Agent A 容器
独立浏览器
独立文件系统
独立网络栈
Agent B 容器
独立浏览器
独立文件系统
独立网络栈
宿主机（共享硬件）
沙盒配置示例（openclaw.json） 
{
  agents: {
    list: [
      {
        id: "agent-a",
        sandbox: {
          mode: "all",      // 启用沙盒
          scope: "agent",   // 每个 Agent 独立容器
        }
      },
      {
        id: "agent-b",
        sandbox: {
          mode: "all",
          scope: "agent",
        }
      }
    ]
  }
}

实际建议

如果你的场景是：多个 Agent 都需要浏览器自动化

方案 复杂度 隔离性 适用场景
独立浏览器实例（不同 userDataDir） 低 中 简单网页操作
虚拟桌面（Windows 多桌面/VNC） 中 高 需要可视化操作
Docker 容器 中 高 服务器部署
多台机器/VM 高 最高 生产环境
独立浏览器配置示例（Python） 
def capture_with_playwright(url, output_path, agent_id="default", wait_time=5):
    """Capture screenshot using Playwright with agent isolation."""
    user_data_dir = f"/tmp/playwright-{agent_id}"  # 每个 Agent 独立目录
    
    with sync_playwright() as p:
        browser = p.chromium.launch_persistent_context(
            user_data_dir=user_data_dir,  # 隔离用户数据
            headless=True,
            viewport={'width': 1920, 'height': 1080}
        )
        page = browser.new_page()
        page.goto(url, wait_until='networkidle')
        page.screenshot(path=output_path, full_page=True)
        browser.close()

总结

多个 Agent 任务是否并发？ 是的，默认并发执行
会冲突吗？ 取决于资源：文件系统一般OK，浏览器/屏幕会冲突
如何解决？ 资源隔离（独立实例/沙盒/容器）或任务队列
OpenClaw 内置支持吗？ 有沙盒模式，但浏览器自动化需要自己配置隔离
一句话总结：多 Agent 是并发的，但共享资源需要你自己做好隔离设计。
