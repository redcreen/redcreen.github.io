🧠 OpenClaw 多实例架构

深入理解 Multi-Agent Routing 的设计与实现

核心概念

🖥️

Gateway

一个网关进程，管理所有消息通道（WhatsApp、Telegram、Discord 等）

🧠

Agent

一个独立的"大脑"，有自己的工作区、配置文件、会话历史

👤

Account

一个通道账号（如一个 WhatsApp 手机号、一个 Discord Bot）

🔗

Binding

路由规则，决定消息交给哪个 Agent 处理

↓

配置文件示例

~/.openclaw/openclaw.json 
{
  // 定义多个 Agent
  agents: {
    list: [
      {
        id: "personal",
        name: "个人助手",
        workspace: "~/.openclaw/workspace-personal",
        agentDir: "~/.openclaw/agents/personal/agent",
        default: true
      },
      {
        id: "work",
        name: "工作助手",
        workspace: "~/.openclaw/workspace-work",
        agentDir: "~/.openclaw/agents/work/agent",
        model: "anthropic/claude-sonnet-4-5"
      }
    ]
  },
  
  // 路由绑定规则
  bindings: [
    // WhatsApp 个人号 → personal 代理
    { 
      agentId: "personal", 
      match: { channel: "whatsapp", accountId: "personal" } 
    },
    // WhatsApp 工作号 → work 代理
    { 
      agentId: "work", 
      match: { channel: "whatsapp", accountId: "work" } 
    }
  ],
  
  channels: {
    whatsapp: {
      accounts: {
        personal: {},
        work: {}
      }
    }
  }
}

常用命令

openclaw agents list --bindings 查看所有代理
openclaw agents add work 添加新代理
openclaw channels login --channel whatsapp --account personal 登录多个 WhatsApp 账号
openclaw gateway restart 重启网关应用配置

关键特性总结

完全隔离
每个 Agent 有自己的工作区、会话、记忆、认证
独立配置
每个 Agent 可以用不同的模型、工具权限、沙盒设置
灵活路由
按通道、账号、群组、个人等多种方式路由
独立人格
每个 Agent 有自己的 SOUL.md / AGENTS.md
